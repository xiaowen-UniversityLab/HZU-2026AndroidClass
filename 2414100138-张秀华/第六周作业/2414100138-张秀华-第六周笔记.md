Android 平台支持三大类传感器：

1.运动传感器
这类传感器测量三个轴向上的加速力和旋转力。这个类别中包含加速度计、重力传感器、陀螺仪和旋转矢量传感器。

重力传感器：sensor = sensorManager.getDefaultSensor(Sensor.TYPE_GRAVITY);
加速度计：sensor = sensorManager.getDefaultSensor(Sensor.TYPE_LINEAR_ACCELERATION);
旋转矢量传感器：sensor = sensorManager.getDefaultSensor(Sensor.TYPE_ROTATION_VECTOR);
陀螺仪：sensor = sensorManager.getDefaultSensor(Sensor.TYPE_GYROSCOPE);

重大运动传感器：sensor = sensorManager.getDefaultSensor(Sensor.TYPE_SIGNIFICANT_MOTION);
private TriggerEventListener triggerEventListener;
triggerEventListener = new TriggerEventListener() {

    }
};
sensorManager.requestTriggerSensor(triggerEventListener, mSensor);

2.环境传感器
这类传感器测量各种环境参数，如环境气温、气压、照度和湿度。这个类别中包含气压计、光度计和温度计。四个环境传感器均基于硬件 并且只有当设备制造商内置到设备中时才可用，在运行时验证是否存在环境传感器 

3.位置传感器
这类传感器测量设备的物理位置。这个类别中包含屏幕方向传感器和磁力计。


4.SensorManager
可以使用这个类来创建传感器服务的实例。该类提供了各种方法来访问和列出传感器，注册和取消注册传感器事件监听器，以及获取屏幕方向信息。该类还提供了几个传感器常量，用于报告传感器精确度、设置数据采集频率和校准传感器。

4.1SensorManager对象调用registerListener注册监听器：记得一定要实现接口implements SensorEventListener！！！！！！！！！！！！！！！！
为传感器注册监听事件，通过调用sensorManager对象的registerListener（）方法来注册监听事件。
第一个参数是上下文对象，第二个是传感器对象，第三个是传感器的延时时间的精密程 度，越精密越耗电。
SENSOR_DELAY_FASTEST——延时：0ms
SENSOR_DELAY_GAME——延时：20ms
SENSOR_DELAY_UI——延时：60ms
SENSOR_DELAY_NORMAL——延时：200ms
如：sensorManager.registerListener(MainActivity.this,mLight,SensorManager.SENSOR_DELAY_FASTEST);
监听器取消注册：
sensorManager.unregisterListener(this);

5.Sensor
您可以使用这个类来创建特定传感器的实例。此类提供了各种方法来确定传感器的特性。
6.SensorEvent
系统使用以下类来创建传感器事件对象，该对象提供有关传感器事件的信息。传感器事件对象中包含以下信息：原始传感器数据、生成事件的传感器类型、数据的准确度和事件的时间戳。

7.SensorEventListener
可以使用此接口创建两种回调方法，以在传感器值或传感器精确度发生变化时接收通知（传感器事件）。用类实现这个接口：implements SensorEventListener
重写两个通过 SensorEventListener 接口公开的回调方法：onAccuracyChanged() 和 onSensorChanged()。Android 系统会在发生以下情况时调用这两个方法：
7.1：onSensorChanged(SensorEvent event);该方法在传感器的值发生改变的时候调用，其参数是一个SensorEvent 对象，通过该对象的values属性可以获取传感器的值，该值是一个数组，该变量最多有三个元素，而且传感器不同，对应元素代表的含义也不同
7.2：onAccuracyChanged(Sensor sensor , int accuracy);当传感器的进度发生改变时会回调，
参数说明：
sensor：传感器对象
accuracy：表示传感器新的精度

8.在清单文件中使用 <uses-feature> 元素，以对不具备适合应用的传感器配置的设备屏蔽的应用
<uses-feature android:name="android.hardware.sensor.accelerometer"
              android:required="true" />
仅当的应用完全依赖于特定传感器时，才应将描述符设置为 android:required="true"。如果的应用使用某种传感器来实现某项功能，但没有该传感器也可以运行，那么就应该在 <uses-feature> 元素中列出该传感器，但将描述符设置为 android:required="false"。这样可以确保设备即使没有该传感器也可以安装的应用。这也是帮助跟踪应用所使用的功能的最佳项目管理方法。请记住，如果应用使用特定的传感器，但没有该传感器也能运行，那么就应该在运行时检测传感器，并根据需要停用或启用应用功能。

9.将传感器数据对应到屏幕显示，需要使用 getRotation() 方法确定屏幕的旋转度，然后使用 remapCoordinateSystem() 方法将传感器坐标映射到屏幕坐标。



分区存储：
1.应用自己的私有目录：随便读写

1.1用：
获取私有文件夹目录
getExternalFilesDir(Environment.DIRECTORY_PICTURES)
直接创建文件 / 文件夹
new File(文件夹目录, "文件名")
直接读写
不需要权限，不会报错，卸载自动删除

1.2如写入图片：
获取目录
File sandboxDir = getExternalFilesDir(Environment.DIRECTORY_PICTURES);
在目录里创建文件
File myFile = new File(sandboxDir, "test.png");
直接写入文件（不需要任何权限！）
try (FileOutputStream fos = new FileOutputStream(myFile)) {
    // 比如把图片写进去
    bitmap.compress(Bitmap.CompressFormat.PNG, 100, fos);
    Toast.makeText(this, "保存成功", Toast.LENGTH_SHORT).show();
} catch (IOException e) {
    e.printStackTrace();
}


2.系统公共目录（Pictures、Download、DCIM…）：
2.1图片 / 视频 / 音乐 → 走 用 MediaStore 访问公共媒体目录，如：想把图片保存到 系统相册里的 Pictures，让用户在相册能看到。
2.2下载 / 文档 / 其他文件 → 走 SAF 文件选择器，如：想读写 Download 里的文件，必须弹出系统文件选择器让用户授权（SAF）

2.3用：
ContentValues 告诉系统文件名、路径
getContentResolver().insert() 让系统创建文件
拿到 Uri 后用流写入
系统自动帮你放到对应的文件夹
保存到 Pictures → 不需要权限
保存到 Download → 不需要权限
只需要在 AndroidManifest.xml 加两行声明:
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
只有读取图片才需要动态申请权限
// 检查是否有存储权限

2.4如保存文件用法：
ContentValues values = new ContentValues();
values.put(MediaStore.Downloads.DISPLAY_NAME, "mydoc.txt");
values.put(MediaStore.Downloads.MIME_TYPE, "text/plain");
values.put(MediaStore.Downloads.RELATIVE_PATH, "Download/");

Uri uri = getContentResolver().insert(MediaStore.Downloads.EXTERNAL_CONTENT_URI, values);

try (OutputStream os = getContentResolver().openOutputStream(uri)) {
    String content = "你好，这是分区存储保存的文件";
    os.write(content.getBytes());
    Toast.makeText(this, "保存到 Download 成功", Toast.LENGTH_SHORT).show();
} catch (IOException e) {
    e.printStackTrace();
}
