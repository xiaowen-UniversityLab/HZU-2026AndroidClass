1.绑定相机生命周期之前要先解绑之前的provider.unbindAll();
2.  Activity 启动 → 自动打开相机
    Activity 暂停（比如切后台）→ 自动关闭相机，省电量
    Activity 销毁 → 自动释放相机资源，不会内存泄漏

3.startCamera()
获取相机管家 ProcessCameraProvider（异步）
配置三个用例：
    Preview → 显示预览画面（连 PreviewView）
    ImageCapture → 拍照
    VideoCapture → 录像
选择后置摄像头
解绑旧用例 + 绑定新用例到Activity生命周期

4.拍照之前要确认初始化的startCamera() 里的拍照功能不为空，不然会闪退
5.照片保存要指定保存路径：
    Android 9 以上：保存在 外部存储/Pictures/CameraX-Image 目录下。
    Android 9 以下：不用手动写路径，系统自动处理（因为老版本对目录限制少）
        if (Build.VERSION.SDK_INT > Build.VERSION_CODES.P) {
            contentValues.put(MediaStore.Images.Media.RELATIVE_PATH, "Pictures/CameraX-Image");
        }
6.执行takePhoto()
    ↓
校验imageCapture是否为空（防崩溃）
    ↓
生成带时间戳的唯一文件名
    ↓
创建ContentValues，填写照片名、格式、保存路径
    ↓
打包成OutputFileOptions保存配置
    ↓
调用takePicture()执行拍照
    ↓
相机拍摄当前预览画面，按配置保存到相册
    ↓
回调结果：
    成功 → 弹Toast显示照片URI
    失败 → 弹Toast提示失败

7.Android 10 及以下老设备，不能同时开「预览 + 拍照 + 图像分析」三个用例，会冲突。
8.拿到摄像头每一帧实时预览的画面，在后台做自定义处理（比如算亮度、人脸识别、二维码扫描、AI 识别等），不影响预览和拍照。
即要做图像分析，必须写一个类，实现 ImageAnalysis.Analyzer 接口，CameraX 会把摄像头拍到的每一帧画面，自动传给这个类的 analyze() 方法，在方法里写处理逻辑就行。
9.检查录音权限
    if (PermissionChecker.checkSelfPermission(this, Manifest.permission.RECORD_AUDIO)== PermissionChecker.PERMISSION_GRANTED) {
            pendingRecording = pendingRecording.withAudioEnabled();
        }
10.用户点击「录像按钮」
    ↓
执行 captureVideo()
    ↓
校验 videoCapture 是否为空（防崩溃）
    ↓
检查是否正在录制：
   正在录制 → 停止录制，重置按钮，结束，不会执行后面的start
   未录制 → 继续
    ↓
生成带时间戳的唯一文件名
    ↓
创建 ContentValues，填写视频名、格式、保存路径
    ↓
打包成 MediaStoreOutputOptions 保存配置
    ↓
准备录制（PendingRecording），开启音频（有权限的话）
    ↓
调用 start() 正式开始录制，绑定回调
    ↓
录制过程中：
   - 按钮显示「停止录制」
   - 摄像头实时录制画面+音频
    ↓
用户点击停止 / 录制出错：
   - 触发 Finalize 事件
   - 保存视频到相册
   - 弹出成功/失败提示
   - 重置按钮为「开始录制」

11.拍照完成后，结果会回调，如果要更新ui、toast这些，要用ContextCompat.getMainExecutor(this)指定在主线程执行。cameraExecutor创建的单线程池主要用在ImageAnalysis 的 setAnalyzer ()分析图片上，如：imageAnalysis.setAnalyzer(cameraExecutor, analyzer);
拍照 / 录像 / 预览不用传cameraExecutor，CameraX 自动管理内部线程
12.LuminosityAnalyzer 是给 CameraX 自动调用的，不是手动调用的！不能自己 new 出来，然后自己调用 analyze ()！analyze () 方法需要 ImageProxy，是CameraX 提供的，它会会自动在后台线程调用 analyze ()
13.cameraExecutor用法：
cameraExecutor.execute(() -> {
    // 1. 这里是后台线程：做耗时计算，如：
    int result = 1000 + 2000;

    // 2. 要更新 UI  必须切回主线程
    runOnUiThread(() -> {
        Toast.makeText(this, "计算完成：" + result, Toast.LENGTH_SHORT).show();
    });
});
