1.在gradle里面配置ViewBinding
 buildFeatures{
        viewBinding = true
    }

绑定类会在编译时自动生成，位于 build/generated/data_binding_base_class_source_out 目录下

2.
//声明全局变量
private ActivityMainBinding binding;
//绑定对象
binding = ActivityMainBinding.inflate(getLayoutInflater());
//设置内容视图
setContentView(binding.getRoot());
//通过绑定对象，可以直接访问布局文件中的视图控件。
binding.tv1.setText("修改后");
binding.btn1.setOnClickListener(...)

具体解释：
2.1调用 getLayoutInflater 方法获取当前Activity的 LayoutInflater 实例。
2.2使用这个 LayoutInflater 实例调用 ActivityMainBinding.inflate 方法，这个方法是由 ViewBinding 功能自动生成的一个静态方法。它用于创建 ActivityMainBinding 实例。这个方法会解析布局文件 activity_main.xml，并返回一个绑定对象，通过这个对象可以访问布局中的所有视图。
2.3ActivityMainBinding 实例会包含对 activity_main.xml 布局文件中所有视图的引用。通过这个绑定对象，你可以直接访问布局文件中的视图，而无需使用 findViewById 方法。
2.4使用setContentView将布局文件加载到当前活动中时，通过binding.getRoot()获取布局资源ID


Retrofit：
1.Retrofit将okhttp请求抽象成java接口，采用注解描述和配置网络请求参数
2.retrofit把网络请求的Url分成两部分设置：第一部分在创建Retrofit实例时通过.baseUrl()设置，第二部分在网络接口注解中设置，比如上面接口的@GET("user")注解括号中的内容user，网络请求的完整地址Url = Retrofit实例.baseUrl()+网络请求接口注解()。

在mainfest中申请权限
<uses-permission android:name="android.permission.INTERNET"/>
添加依赖：
implementation 'com.squareup.retrofit2:retrofit:2.5.0'
implementation 'com.squareup.retrofit2:converter-gson:2.0.2'

3.GET请求：

3.1有限个请求参数的GET请求
 @GET("user")
 Call<ResponseBody> getData2(@Query("id") long idLon, @Query("name") String nameStr);

GET请求参数用@Query进行注解，后面idLon的取值作为"id"的值，相当于键值对 ，Retrofit会把两个字段拼接到接口中，追加到"/user"后面。
如：baseUrl为https://api.github.com/，那么拼接网络接口注解中的地址后变为：https://api.github.com/user，我们需要传入的id=10006，name="哈哈哈"，那么拼接参数后就是完整的请求地址：https://api.github.com/user?id=10006&name=哈哈哈。

3.2如果有不确定的把表单参数可以使用@QueryMap注解来添加参数，通过Map将不确定的参数存入，然后在方法体中带给网络接口，相当于多个Query参数，@QueryMap Map<String, Object> map

4.POST请求

4.1 Post请求如果有参数需要在头部添加@FormUrlEncoded注解，表示请求实体是一个From表单，每个键值对需要使用@Field注解，这是发送Post请求时，提交请求的表单字段，必须要将@FormUrlEncoded和@Field配合使用。若没有参数则只需要@POST即可
如：
@FormUrlEncoded
@POST("user/emails")
Call<ResponseBody> getPostData2(@Field("name") String nameStr, @Field("sex") String sexStr);

4.2 @FieldMap Map<String, Object> map：与@Field作用一致，@FieldMap 用于有多个不确定参数时，通过Map将不确定的参数传入，相当于多个Field参数

4.3 @Body：可以传递自定义类型数据给服务器，多用于post请求发送非表单数据，比如用传递Json 格式数据

4.4 @Body注解不能用于表单或者支持文件上传的表单的编码，即不能与@FormUrlEncoded和@Multipart注解同时使用，否则会报错！！！！！！

5.@Url: 将地址以参数的形式传入即可。如果有@Url注解时，GET括号传入的Url可以省略。

6.使用流程：
创建接受服务器返回数据的类：接收网络请求返回的数据
创建用于描述网络接口的接口：用注解描述和配置网络请求参数，封装Url地址和网络数据请求
创建Retrofit实例：
    ①通过new Retrofit.Builder().buile()构建Retrofit实例对象，同时设置baseUrl地址，注意：baseUrl地址必须以“/”结尾，否则会报错
    ②通过addConverterFactory设置数据解释器，这里添加的是Gson解释器。这是为了使来自接口的json结果会自动解析成定义好的字段和类型都相符的json对象接受类
创建网络请求接口实例：调用Retrofit实例对象create()方法，传入网络接口，得到接口对象实例，调用接口类中的方法。
发送网络请求（同步/异步）
处理返回的数据

注意！网络请求回调拿回来的bean转成list是会创建一个新的list！ Adapter 初始化时绑定的是旧的空 List，notifyDataSetChanged() 只会刷新 Adapter 绑定的那个 旧的List，而 mBeanList 指向了新的 List，所以 Adapter 永远拿不到新数据！可以用addAll把新的list添加到Adapter绑定的list中去


OkHttp3中的拦截器

每个拦截器都持有一个拦截链（Interceptor.Chain）对象，通过调用chain.proceed(request)将请求传递给下一个拦截器。

1.应用拦截器：主要用于应用级的处理，它们在整个请求过程的最外层。应用拦截器可以添加、删除或修改请求和响应，甚至可以短路请求，直接返回自定义响应。
2.网络拦截器：用于在网络层处理请求和响应，它们只能处理真正发出的请求和接收到的响应。网络拦截器无法短路请求，也无法影响缓存的处理。

3.实现拦截器：实现Interceptor接口并重写intercept方法
4.在OkHttpClient中添加拦截器：通过addInterceptor方法添加应用拦截器，通过addNetworkInterceptor方法添加网络拦截器。
如：OkHttpClient client = new OkHttpClient.Builder()
    .addInterceptor(new LoggingInterceptor())
    .addNetworkInterceptor(new StethoInterceptor())
    .build();



JSON:
要使用GSON进行数据转换，首先需要创建一个 Gson 实例。然后，可以通过该实例调用 toJson() 方法将Java对象转换为JSON字符串，或者使用 fromJson() 方法将JSON字符串转换回Java对象

在gradle添加依赖：

1.创建gson实例：Gson gson = new Gson();

2.java转json：  Person person = new Person("张三", 30);
                String json = gson.toJson(person);

3.json转java对象：   Person parsedPerson = gson.fromJson(json, Person.class);
                    System.out.println(parsedPerson.getName()); 

可以在retrofit创建的时候加上.addConverterFactory(GsonConverterFactory.create())，会把请求返回的json数据转成对应的javabean对象（前提是要有这个JavaBean对象！）
若要根据原始json数据编写JavaBean的话，可以把网络接口暂时写成返回为ResponsBody类型的，在retrofit调用返回处用String json = response.body().string();查看原始json数据
后续有JavaBean对象后，把返回值改成对应JavaBean对象即可
