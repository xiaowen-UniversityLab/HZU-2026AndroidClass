1.WindowManager的LayoutParams的type要根据sdk设置不同的，如下所示：
        if (Build.VERSION.SDK_INT >= 26) {
            mWindowParams.type = WindowManager.LayoutParams.TYPE_APPLICATION_OVERLAY;
        } else {
            mWindowParams.type = WindowManager.LayoutParams.TYPE_SYSTEM_ALERT;
        }
2.在 AndroidManifest.xml 中必须添加以下权限：
<uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW"/>

3.在 Android 中，涉及 Dialog、PopupWindow、悬浮窗 等 UI 控件的创建，必须使用 Activity 的 Context（this），不要用 getApplicationContext()，不然会报 Token 错误

4.悬浮窗权限申请
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
        if (!Settings.canDrawOverlays(this)) {
            // 无权限：跳转到系统设置页，让用户手动开启「显示在其他应用上层」
            Intent intent = new Intent(Settings.ACTION_MANAGE_OVERLAY_PERMISSION,
                    Uri.parse("package:" + getPackageName()));
            startActivity(intent);
            Toast.makeText(this, "请开启权限", Toast.LENGTH_LONG).show();
            return; // 没权限，直接return，不启动服务
        }
    }


5.自定义 View / 布局：可以自定义悬浮窗的 UI（比如 Service 悬浮窗的展开 / 收起按钮、文本内容）
①通过 LayoutInflater 加载布局，如：
    private View rootLayout;
    rootLayout = LayoutInflater.from(mContext).inflate(R.layout.floatingwidow_in_service, null);
②通过findViewById 绑定控件，如：
    btn_hide = rootLayout.findViewById(R.id.fw_in_service_hide);
6.动态更新：可以通过mWindowManager.updateViewLayout动态修改悬浮窗的大小、位置
7.在ontouch中实现悬浮窗的拖拽：
拖拽公式：悬浮窗左上角坐标 = 手指当前屏幕坐标 - 手指按下时在悬浮窗内的偏移量，如：mWindowParams.x = (int) (mInScreenX - mInViewX) 相当于motionEvent.getRawX()-motionEvent.getX()
8.保证整个 APP 永远只有一个悬浮窗，不会重复创建，不会乱弹窗：用单例模式：私有的构造方法，对外提供一个方法来获取悬浮窗对象，如下：
    //私有静态实例，私有构造方法
    private static LyricWindow INSTANCE;
    private LyricWindow(Context context) {
        mContext = context;
        initLyricWindow();
        isInit = true;
    }
    //获取实例方法
    public static synchronized LyricWindow getInstance(Context context) {
        if (INSTANCE == null) {
            INSTANCE = new LyricWindow(context.getApplicationContext());
        }
        return INSTANCE;
    }
