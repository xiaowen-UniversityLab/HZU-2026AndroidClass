1.App 刚打开时亮度检测是准的，但过了一秒钟，无论怎么遮挡摄像头，提示文字都不动了
原因：你占用了相机的“餐盘”（缓冲区）却没还回去，导致相机没法发新图。
在 analysis.setAnalyzer里：
加上image.close(); 就可修复

2.App启动崩溃，一点击图标，App还没看到预览画面就闪退了。
原因：你还没拿到用户的“许可证”（摄像头权限）就急着去启动相机。
onCreate 里的权限判断，给予PERMISSION_GRANTED
if (ContextCompat.checkSelfPermission(this, Manifest.permission.CAMERA) == PackageManager.PERMISSION_GRANTED) {
    startCamera(); 
}

3. 亮度检测非常卡顿
原因：让主线程去干计算亮度的活，导致主线程没空去更新界面。
用专门的cameraExecutor

analysis.setAnalyzer(cameraExecutor, image -> { ... });

4. 界面更新“非法操作”故障
现象：代码逻辑没问题，但在更新文字时，App 突然崩溃，报错：CalledFromWrongThreadException。

原因：后台线程试图直接动手修改 UI 控件，这在 Android 不行

runOnUiThread(() -> tvLowLight.setVisibility(...)); // 必须通知然后回主线程修改 UI
