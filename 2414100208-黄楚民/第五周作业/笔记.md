1.防止你的应用在用户接电话或听音乐时多个应用同时出声，必须遵循系统的音频调度规则。

申请焦点： 播放前通过 AudioManager.requestAudioFocus() 申请。

监听变化：

AUDIOFOCUS_LOSS：永久失去焦点（如打开了别的播放器），你应该停止播放并释放资源。

AUDIOFOCUS_LOSS_TRANSIENT：暂时失去（如来电），你应该暂停。

AUDIOFOCUS_GAIN：重新获得焦点，可以恢复播放。

2.Activity 配置： 在 AndroidManifest.xml 中为 Activity 设置 android:screenOrientation="landscape"（横屏模式）。

3.没有画面

原因：没有setDisplay，如果不写setDisplay，会只听的到视频的声音，但屏幕是黑的

mediaPlayer.setDisplay(holder); 

4.进度条拖动困难

原因：忘记判断 fromUser

@Override
public void onProgressChanged(SeekBar seekBar, int progress, boolean fromUser) {
    if (fromUser && mediaPlayer != null) { 
        mediaPlayer.seekTo(progress);
    }
}这样你手动拖动时，进度条就能被精确拖动，因为进度条的变化有两种：一种是视频播放自己涨的，一种是人手滑动的。如果不加 fromUser 判断，代码会自动进入死循环。
