1.老是忘了在 Java 中，类名首字母大写（如 String, Manifest, Intent），变量名首字母小写（如 intent, permissions）。

2.如果在变量名前加了类型，它就变成了局部变量，出了大括号就消失了。正确做法：在类名下方定义（不赋值），在方法里直接使用（赋值）。

3.成对原则，不能只写了 registerReceiver，忘了写 unregisterReceiver。。如果在 onResume 注册，必须在 onPause 注销；如果在 onCreate 注册，必须在 onDestroy 注销。

4.查询照片时 App 闪退，或者多次运行后系统内存溢出。直接调用 cursor.getLong() 而不先判断 cursor.moveToFirst()，或者用完后忘了 cursor.close()。

5.只写了 Java 里的动态请求代码，忘了在 AndroidManifest.xml 里写 <uses-permission>。权限必须双向登记。XML 是“备案”，Java 代码是“当面问用户要”。缺一不可。

6.查询结果永远为空，哪怕相册里有照片。URI 填错，或者在 Android 13+ 系统上用了老版本的权限名。照片地址是 MediaStore.Images.Media.EXTERNAL_CONTENT_URI。
Android 13 (API 33) 之后，权限从 READ_EXTERNAL_STORAGE 变成了 READ_MEDIA_IMAGES。
